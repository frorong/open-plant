1. WsiViewer: composition root and child layers

WsiViewer is the center of the current React API. It owns tile rendering, shared overlay canvases, and camera state. Points, ROI, drawing, and heatmap are attached as child layers. 

import { useState } from "react";
import {
  DrawingLayer,
  HeatmapLayer,
  OverlayLayer,
  PatchLayer,
  PointLayer,
  RegionLayer,
  WsiViewer,
  type DrawOverlayShape,
  type DrawTool,
  type HeatmapPointData,
  type PointSizeByZoom,
  type WsiImageSource,
  type WsiPointData,
  type WsiRegion,
  type WsiViewState,
} from "open-plant";

function ViewerShell({
  source,
  pointData,
  palette,
  heatmapData,
}: {
  source: WsiImageSource | null;
  pointData: WsiPointData | null;
  palette: Uint8Array | null;
  heatmapData: HeatmapPointData | null;
}) {
  const [viewState, setViewState] = useState<Partial<WsiViewState>>({});
  const [drawTool, setDrawTool] = useState<DrawTool>("cursor");
  const [regions, setRegions] = useState<WsiRegion[]>([]);

  const pointSizeByZoom: PointSizeByZoom = { 1.25: 2, 2.5: 3, 5: 4, 10: 6, 20: 8 };
  const overlayShapes: DrawOverlayShape[] = [
    {
      id: "focus-mask",
      coordinates: [[[1000, 1000], [2000, 1000], [2000, 2000], [1000, 2000], [1000, 1000]]],
      closed: true,
      invertedFill: { fillColor: "rgba(0, 0, 0, 0.14)" },
    },
  ];

  return (
    <WsiViewer
      source={source}
      viewState={viewState}
      onViewStateChange={setViewState}
      zoomSnaps={[1.25, 2.5, 5, 10, 20, 40]}
      zoomSnapFitAsMin
      onPointerWorldMove={(event) => console.log(event.coordinate)}
      onStats={(stats) => console.log(stats.frameMs, stats.visible)}
      style={{ width: "100%", height: 720 }}
    >
      <PointLayer data={pointData} palette={palette} sizeByZoom={pointSizeByZoom} />
      <HeatmapLayer data={heatmapData} visible opacity={0.7} radius={2} blur={3} />
      <RegionLayer regions={regions} />
      <DrawingLayer
        tool={drawTool}
        onComplete={(result) => {
          setRegions((prev) => [
            ...prev,
            { id: crypto.randomUUID(), label: "ROI", coordinates: result.coordinates },
          ]);
          setDrawTool("cursor");
        }}
      />
      <OverlayLayer shapes={overlayShapes} />
      <PatchLayer regions={regions.filter((region) => String(region.label).startsWith("Patch"))} />
    </WsiViewer>
  );
}
WsiViewerCanvas is not exported from the current package. New implementations should treat WsiViewer + child layers as the primary path.
Key exportsRole
WsiViewer, WsiViewerPropsRoot for camera, tiles, and shared canvases.
PointLayer, PointLayerPropsPass point buffers and palette into the renderer.
RegionLayer, RegionLayerPropsShow persisted ROI, hover, active state, and labels.
DrawingLayer, DrawingLayerPropsControl draw tools and receive completed shapes.
PatchLayer, OverlayLayerRender patch-only polygons and custom overlay shapes.
PointerWorldMoveEvent, WsiRenderStats, WsiTileErrorEventTyped payloads for pointer, stats, and tile errors.

2. normalizeImageInfo / normalizeImageClasses / toTileUrl

Fetch image metadata from your API, then normalize it into runtime inputs with normalizeImageInfo and normalizeImageClasses. Use toBearerToken so your fetch path accepts either raw or prefixed tokens. 

import {
  normalizeImageClasses,
  normalizeImageInfo,
  toBearerToken,
  toTileUrl,
  type RawImagePayload,
  type WsiClass,
  type WsiImageSource,
} from "open-plant";

const imageId = "69898975a05a9cd9ec6fe311";
const token = toBearerToken("<access-token>");

const response = await fetch(`/api/v4/images/${imageId}/info`, {
  headers: { Authorization: token },
});
if (!response.ok) throw new Error(`HTTP ${response.status}`);

const raw = (await response.json()) as RawImagePayload;
const source: WsiImageSource = normalizeImageInfo(raw, "https://your-cdn.example.com/ims");
const classes: WsiClass[] = normalizeImageClasses(raw);
const previewTileUrl = toTileUrl(source, source.maxTierZoom, 0, 0);
RawImagePayload, RawImsInfo, and RawWsiClass describe the server response shape. WsiImageSource, WsiClass, WsiImageColorSettings, and WsiViewState are the runtime shapes your app usually stores.

3. PointLayer: point rendering and picking

Point decoding is outside the library. The core contract is to build WsiPointData with TypedArrays, then pass the palette from buildClassPalette into PointLayer. 

import { useMemo, useRef } from "react";
import {
  DEFAULT_POINT_COLOR,
  PointLayer,
  buildClassPalette,
  hexToRgba,
  type PointHoverEvent,
  type PointQueryHandle,
  type PointSizeByZoom,
  type WsiClass,
  type WsiPointData,
} from "open-plant";

function CellLayer({
  classes,
  positions,
  paletteIndices,
}: {
  classes: WsiClass[];
  positions: Float32Array;
  paletteIndices: Uint16Array;
}) {
  const pointRef = useRef<PointQueryHandle>(null);
  const palette = useMemo(() => buildClassPalette(classes), [classes]);

  const pointData: WsiPointData = {
    count: positions.length / 2,
    positions,
    paletteIndices,
  };

  const sizeByZoom: PointSizeByZoom = { 1.25: 2, 2.5: 3, 5: 4, 10: 5, 20: 7 };

  console.log(DEFAULT_POINT_COLOR, hexToRgba("#13bfa0"));

  return (
    <PointLayer
      ref={pointRef}
      data={pointData}
      palette={palette.colors}
      sizeByZoom={sizeByZoom}
      strokeScale={1.15}
      innerFillOpacity={0.2}
      onHover={(event: PointHoverEvent) => console.log(event.id, event.coordinate)}
      onClick={(event) => console.log(event.index, event.id)}
    />
  );
}

const hit = pointRef.current?.queryAt([12000, 18000]);
If all you need is coordinate picking, PointQueryHandle plus onHover/onClick is enough. Drop down to the Point Hit Index APIs only when you need direct control over the spatial hash.

4. DrawingLayer / RegionLayer / PatchLayer / OverlayLayer

DrawingLayer is the normal path for user-created ROI. If coordinates already exist, RegionLayer, PatchLayer, and OverlayLayer are enough. 

import {
  DrawingLayer,
  OverlayLayer,
  PatchLayer,
  RegionLayer,
  closeRing,
  createCircle,
  createRectangle,
  type BrushOptions,
  type DrawAreaTooltipOptions,
  type DrawResult,
  type DrawTool,
  type PatchDrawResult,
  type StampOptions,
} from "open-plant";

const manualRectangle = createRectangle([1200, 1400], [2600, 2800]);
const manualCircle = createCircle([3200, 3200], [3800, 3800]);
const manualPolygon = closeRing([[4200, 4200], [5200, 4400], [4900, 5300]]);

const brushOptions: BrushOptions = {
  radius: 24,
  edgeDetail: 1.5,
  edgeSmoothing: 2,
  clickSelectRoi: true,
};

const stampOptions: StampOptions = {
  rectangleAreaMm2: 2,
  circleAreaMm2: 0.2,
  rectanglePixelSize: 4096,
};

const areaTooltip: DrawAreaTooltipOptions = { enabled: true };
const drawTool: DrawTool = "freehand";

<RegionLayer
  regions={[
    { id: "rect", label: "Rect ROI", coordinates: manualRectangle },
    { id: "circle", label: "Circle ROI", coordinates: manualCircle },
    { id: "poly", label: "Manual ROI", coordinates: manualPolygon },
  ]}
  strokeStyle={{ color: "#ffd166", width: 2.5 }}
  hoverStrokeStyle={{ color: "#ff6a3d", width: 3 }}
  activeStrokeStyle={{ color: "#ff2f2f", width: 3.2, shadowBlur: 12 }}
  labelStyle={({ zoom }) => ({ fontSize: zoom > 4 ? 13 : 11 })}
  autoLiftLabelAtMaxZoom

<DrawingLayer
  tool={drawTool}

  areaTooltip={areaTooltip}
  onComplete={(result: DrawResult) => console.log(result.intent, result.coordinates)}
  onPatchComplete={(patch: PatchDrawResult) => console.log(patch.tool, patch.coordinates)}
/>

<PatchLayer regions={[{ id: "patch-1", coordinates: manualRectangle }]} />
<OverlayLayer
  shapes={[
    {
      id: "mask",
      coordinates: [[[1000, 1000], [2000, 1000], [2000, 2000], [1000, 2000], [1000, 1000]]],
      closed: true,
      invertedFill: { fillColor: "rgba(0, 0, 0, 0.15)" },
    },
  ]}
/>
import { DrawLayer, useViewerContext } from "open-plant";

function StandaloneDrawOverlay() {
  const { rendererRef, source } = useViewerContext();
  if (!source) return null;

  return (
    <DrawLayer
      tool="rectangle"
      imageWidth={source.width}
      imageHeight={source.height}
      imageMpp={source.mpp}
      imageZoom={source.maxTierZoom}
      projectorRef={rendererRef}
      onDrawComplete={(result) => console.log(result.coordinates)}
    />
  );
}
DrawLayer is a lower-level overlay primitive. Most apps should stay on DrawingLayer unless they need direct projector control.

5. HeatmapLayer / OverviewMap / useViewerContext

Use HeatmapLayer for hotspot density, and pair OverviewMap with useViewerContext when you need a minimap or explicit coordinate transforms. 

import {
  HeatmapLayer,
  OverviewMap,
  useViewerContext,
  type HeatmapKernelScaleMode,
  type HeatmapPointData,
  type OverviewMapOptions,
} from "open-plant";

const heatmapData: HeatmapPointData = {
  count: hotspots.length / 2,
  positions: hotspots,
  weights,
};

const scaleMode: HeatmapKernelScaleMode = "fixed-zoom";

<HeatmapLayer
  data={heatmapData}
  visible
  opacity={0.75}
  radius={2}
  blur={3}
  gradient={["transparent", "#0ff", "#0f0", "#ff0", "#f00"]}
  scaleMode={scaleMode}
  fixedZoom={6}
  densityContrast={4}
  onStats={(stats) => console.log(stats.visiblePoints, stats.renderMs)}
/>
function ViewerChrome() {
  const { rendererRef, source, worldToScreen, screenToWorld, overviewInvalidateRef } = useViewerContext();
  if (!source) return null;

  const options: Partial<OverviewMapOptions> = {
    position: "bottom-right",
    viewportBorderStyle: "dash",
    interactive: true,
  };

  console.log(worldToScreen(1000, 1000), screenToWorld(200, 120));

  return (
    <OverviewMap
      source={source}
      projectorRef={rendererRef}
      invalidateRef={overviewInvalidateRef}
      options={options}
    />
  );
}
__heatmapLayerInternals is exposed for internal/debug style scenarios. It should not be treated as the default public path for app code.

6. filterPointDataByPolygons*: sync / worker / hybrid-webgpu

You can either pre-process point data with the clip functions directly, or let PointLayer handle clipping by passing clipEnabled and clipMode. 

import {
  filterPointDataByPolygons,
  filterPointDataByPolygonsHybrid,
  filterPointDataByPolygonsInWorker,
  filterPointIndicesByPolygons,
  filterPointIndicesByPolygonsInWorker,
  terminateRoiClipWorker,
  type HybridPointClipOptions,
  type PointClipMode,
  type RoiPolygon,
} from "open-plant";

const polygons: RoiPolygon[] = [roi.coordinates as RoiPolygon];

const syncData = filterPointDataByPolygons(pointData, polygons);
const syncIndices = filterPointIndicesByPolygons(pointData, polygons);

const workerResult = await filterPointDataByPolygonsInWorker(pointData, polygons);
const workerIndices = await filterPointIndicesByPolygonsInWorker(pointData, polygons);

const hybridOptions: HybridPointClipOptions = { bridgeToDraw: true };
const hybridResult = await filterPointDataByPolygonsHybrid(pointData, polygons, hybridOptions);

const clipMode: PointClipMode = "worker";
terminateRoiClipWorker();
<PointLayer
  data={pointData}
  palette={palette.colors}
  clipEnabled
  clipToRegions={regions}
  clipMode="worker"
  onClipStats={(event) => {
    console.log(event.mode, event.durationMs, event.outputCount);
  }}
/>
ModeRecommended useNotes
"sync"Small data, quick debuggingRuns on the main thread.
"worker"Default production pathBest tradeoff for keeping UI responsive.
"hybrid-webgpu"Opt-in after benchmarking real dataUses bbox prefilter before polygon inclusion checks.
The RoiClipWorkerRequest protocol types are only needed if you are building a custom worker bridge yourself. Most apps only need filterPointDataByPolygonsInWorker.

7. buildPointSpatialIndexAsync / lookupCellIndex

PointLayer events are enough for most hover/click cases. Use buildPointSpatialIndexAsync when you need explicit control over coordinate lookup or custom picking. 

import {
  buildPointSpatialIndexAsync,
  lookupCellIndex,
  terminatePointHitIndexWorker,
  type FlatPointSpatialIndex,
} from "open-plant";

const flatIndex: FlatPointSpatialIndex | null = await buildPointSpatialIndexAsync(pointData, source);

if (flatIndex) {
  const cellIndex = lookupCellIndex(flatIndex, 12, 8);
  console.log(flatIndex.cellSize, flatIndex.safeCount, cellIndex);
}

terminatePointHitIndexWorker();
PointHitIndexWorkerRequest and PointHitIndexWorkerResponse are useful only when you want the worker message contract typed in your own bridge layer.

8. parseWkt / toRoiGeometry / computeRoiPointGroups / createSpatialIndex

This is the path for turning WKT strings into ROI coordinates, computing per-ROI class counts, and keeping a bbox search index for fast lookups. 

import {
  computeRoiPointGroups,
  createSpatialIndex,
  parseWkt,
  toRoiGeometry,
  type RoiPointGroupStats,
  type SpatialExtent,
  type SpatialIndexItem,
} from "open-plant";

const parsed = parseWkt("POLYGON ((1200 1400, 2400 1400, 2400 2600, 1200 2600, 1200 1400))");
const roiGeometry = toRoiGeometry(parsed?.coordinates);

const stats: RoiPointGroupStats = computeRoiPointGroups(pointData, regions, {
  paletteIndexToClassId: new Map([
    [1, "negative"],
    [2, "positive"],
  ]),
});

const index = createSpatialIndex<{ id: string }>(16);
const items: SpatialIndexItem<{ id: string }>[] = [
  { minX: 1200, minY: 1400, maxX: 2400, maxY: 2600, value: { id: "roi-1" } },
  { minX: 2600, minY: 3000, maxX: 4200, maxY: 4600, value: { id: "roi-2" } },
];
index.load(items);

const searchExtent: SpatialExtent = [1800, 1800, 3200, 3200];
const candidates = index.search(searchExtent);
RoiPointGroup, RoiClassCount, ParsedWktGeometry, ParsedWktPolygon, and ParsedWktMultiPolygon are the result types for this workflow.

10. TileViewerCanvas / M1TileRenderer / WsiTileRenderer

These APIs are for engine integration and experiments. They are not the default entry point for most React applications, but they are useful when you need direct renderer control. 

import {
  TileViewerCanvas,
  type TileDefinition,
  type ViewState,
} from "open-plant";

const tiles: TileDefinition[] = [
  { id: "tile-0-0", url: "/tiles/0/0_0.webp", bounds: [0, 0, 1024, 1024] },
];

const viewState: Partial<ViewState> = { zoom: 1, offsetX: 512, offsetY: 512 };

<TileViewerCanvas
  imageWidth={8192}
  imageHeight={8192}
  tiles={tiles}
  viewState={viewState}
  style={{ width: 640, height: 480 }}
/>
import {
  TileScheduler,
  WsiTileRenderer,
  getWebGpuCapabilities,
  prefilterPointsByBoundsWebGpu,
  type ScheduledTile,
  type WsiTileRendererOptions,
} from "open-plant";

const rendererOptions: WsiTileRendererOptions = {
  authToken,
  pointSizeByZoom: { 1.25: 2, 5: 4, 20: 7 },
  tileScheduler: { maxConcurrency: 12, maxRetries: 2 },
  onViewStateChange: (next) => console.log(next.zoom),
  onStats: (stats) => console.log(stats.frameMs),
};

const renderer = new WsiTileRenderer(canvas, source, rendererOptions);
renderer.setPointPalette(palette.colors);
renderer.setPointData(pointData);
renderer.setImageColorSettings({ brightness: 0.05, contrast: 0.1, saturation: 0 });
renderer.fitToImage({ duration: 0 });
renderer.zoomBy(1.2, 400, 300);

const scheduledTile: ScheduledTile = {
  key: "6/10/12",
  tier: 6,
  x: 10,
  y: 12,
  bounds: [10240, 12288, 11264, 13312],
  distance2: 0,
  url: "https://your-cdn.example.com/tiles/6/12_10.webp",
};

const scheduler = new TileScheduler({
  authToken,
  onTileLoad: (tile, bitmap) => console.log(tile.key, bitmap.width, bitmap.height),
});
scheduler.schedule([scheduledTile]);

const caps = await getWebGpuCapabilities();
if (caps.supported) {
  const mask = await prefilterPointsByBoundsWebGpu(
    pointData.positions,
    pointData.count,
    new Float32Array([1200, 1400, 2400, 2600]),
  );
  console.log(mask);
}

renderer.destroy();
scheduler.destroy();
import { M1TileRenderer } from "open-plant";

const simpleRenderer = new M1TileRenderer({
  canvas,
  imageWidth: 8192,
  imageHeight: 8192,
  initialViewState: { zoom: 1 },
});

await simpleRenderer.setTiles(tiles);
simpleRenderer.destroy();
M1TileRenderer and TileViewerCanvas are closer to the light tile-only path. Production WSI integration usually means WsiTileRenderer directly or the higher-level WsiViewer.
