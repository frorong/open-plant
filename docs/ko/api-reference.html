`WsiViewer` — 컴포지션 API (v1.4.4)

선언적 레이어 컴포지션 API입니다. 모놀리식 WsiViewerCanvas는 v1.4.4에서 패키지에서 제거되었습니다. WsiViewer는 WebGL/오버레이 캔버스를 관리하고 ViewerContext를 제공하며, 자식 레이어 컴포넌트를 렌더링합니다. 마이그레이션 가이드 참조.

WsiViewer는 화면에 보이는 캔버스를 정확히 2개(WebGL + 2D 오버레이)만 생성합니다. 레이어 컴포넌트는 이 공유 캔버스에 그리며, 필요하면 내부적으로 offscreen 캐시를 유지할 수 있습니다.

레거시 WsiViewerCanvas v1.4.4+ 에서 제거

모놀리식 WsiViewerCanvas 컴포넌트와 WsiViewerCanvasProps는 open-plant v1.4.4부터 패키지에서 export되지 않습니다.

  • 업그레이드: 위 섹션의 WsiViewer + 레이어 조합과 마이그레이션 가이드를 사용하세요.
  • 당분간 레거시 API 유지: "open-plant": "1.4.3" 등으로 버전을 고정하세요(락파일 기준 마지막 WsiViewerCanvas 포함 버전).
  • 과거 Props 표: git 히스토리 및 migration-1.4.0.md 매핑표에 보존됩니다.
ROI term 통계: onRoiPointGroups 대신 앱 코드에서 computeRoiPointGroups()를 호출하세요. customLayers는 직접 대체 없음 — useViewerContext + 호스트 React 오버레이를 사용하세요.

API 안정성 정책 (WS-10)

SemVer: patch = 버그 수정, minor = 하위호환 기능 추가, major = 브레이킹 변경.
Deprecation 수명주기: 대체 API를 먼저 제공하고 문서에서 deprecated 표시 후 최소 두 번의 minor 릴리스 동안 호환을 유지하며, 제거는 다음 major에서만 수행합니다.
Migration 계약: 브레이킹 변경에는 before/after 예시와 체크리스트를 반드시 동반합니다.
Known limitations: 실제 브라우저 상호작용(pan/zoom/draw) E2E 자동화는 아직 CI에서 완전 커버되지 않습니다.
실제 업그레이드 절차와 before/after 예시는 마이그레이션 가이드를 확인하세요.
