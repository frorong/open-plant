1. WsiViewer: composition root and child layers

현재 공개 React API의 중심은 WsiViewer입니다. 타일 렌더링, view state, 공용 overlay canvas는 여기서 관리하고, 포인트/ROI/드로잉/heatmap은 자식 레이어로 붙입니다. 

import { useState } from "react";
import {
  DrawingLayer,
  HeatmapLayer,
  OverlayLayer,
  PatchLayer,
  PointLayer,
  RegionLayer,
  WsiViewer,
  type DrawOverlayShape,
  type DrawTool,
  type HeatmapPointData,
  type PointSizeByZoom,
  type WsiImageSource,
  type WsiPointData,
  type WsiRegion,
  type WsiViewState,
} from "open-plant";

function ViewerShell({
  source,
  pointData,
  palette,
  heatmapData,
}: {
  source: WsiImageSource | null;
  pointData: WsiPointData | null;
  palette: Uint8Array | null;
  heatmapData: HeatmapPointData | null;
}) {
  const [viewState, setViewState] = useState<Partial<WsiViewState>>({});
  const [drawTool, setDrawTool] = useState<DrawTool>("cursor");
  const [regions, setRegions] = useState<WsiRegion[]>([]);

  const pointSizeByZoom: PointSizeByZoom = { 1.25: 2, 2.5: 3, 5: 4, 10: 6, 20: 8 };
  const overlayShapes: DrawOverlayShape[] = [
    {
      id: "focus-mask",
      coordinates: [[[1000, 1000], [2000, 1000], [2000, 2000], [1000, 2000], [1000, 1000]]],
      closed: true,
      invertedFill: { fillColor: "rgba(0, 0, 0, 0.14)" },
    },
  ];

  return (
    <WsiViewer
      source={source}
      viewState={viewState}
      onViewStateChange={setViewState}
      zoomSnaps={[1.25, 2.5, 5, 10, 20, 40]}
      zoomSnapFitAsMin
      onPointerWorldMove={(event) => console.log(event.coordinate)}
      onStats={(stats) => console.log(stats.frameMs, stats.visible)}
      style={{ width: "100%", height: 720 }}
    >
      <PointLayer data={pointData} palette={palette} sizeByZoom={pointSizeByZoom} />
      <HeatmapLayer data={heatmapData} visible opacity={0.7} radius={2} blur={3} />
      <RegionLayer regions={regions} />
      <DrawingLayer
        tool={drawTool}
        onComplete={(result) => {
          setRegions((prev) => [
            ...prev,
            { id: crypto.randomUUID(), label: "ROI", coordinates: result.coordinates },
          ]);
          setDrawTool("cursor");
        }}
      />
      <OverlayLayer shapes={overlayShapes} />
      <PatchLayer regions={regions.filter((region) => String(region.label).startsWith("Patch"))} />
    </WsiViewer>
  );
}
WsiViewerCanvas는 현재 패키지에서 export되지 않습니다. 신규 구현은 WsiViewer + layer 조합으로 가는 것이 기준입니다.
주요 Export역할
WsiViewer, WsiViewerProps카메라/타일/공용 캔버스의 루트.
PointLayer, PointLayerProps포인트 데이터와 palette를 렌더러에 전달.
RegionLayer, RegionLayerProps영속 ROI, hover, active, label 표시.
DrawingLayer, DrawingLayerPropsdraw tool 상태와 결과 수신.
PatchLayer, OverlayLayer보조 polygon / custom overlay 렌더.
PointerWorldMoveEvent, WsiRenderStats, WsiTileErrorEvent상태/에러/포인터 이벤트 payload.

2. normalizeImageInfo / normalizeImageClasses / toTileUrl

API 서버에서 이미지 info를 받은 뒤 normalizeImageInfo, normalizeImageClasses로 런타임 입력으로 바꿉니다. 토큰은 toBearerToken으로 정규화해 두면 좋습니다. 

import {
  normalizeImageClasses,
  normalizeImageInfo,
  toBearerToken,
  toTileUrl,
  type RawImagePayload,
  type WsiClass,
  type WsiImageSource,
} from "open-plant";

const imageId = "69898975a05a9cd9ec6fe311";
const token = toBearerToken("<access-token>");

const response = await fetch(`/api/v4/images/${imageId}/info`, {
  headers: { Authorization: token },
});
if (!response.ok) throw new Error(`HTTP ${response.status}`);

const raw = (await response.json()) as RawImagePayload;
const source: WsiImageSource = normalizeImageInfo(raw, "https://your-cdn.example.com/ims");
const classes: WsiClass[] = normalizeImageClasses(raw);
const previewTileUrl = toTileUrl(source, source.maxTierZoom, 0, 0);
RawImagePayload, RawImsInfo, RawWsiClass는 서버 응답 shape입니다. WsiImageSource, WsiClass, WsiImageColorSettings, WsiViewState는 앱 코드에서 직접 들고 다니는 쪽의 타입입니다.

3. PointLayer: point rendering and picking

포인트 로더는 라이브러리 밖에서 구현합니다. 핵심은 WsiPointData를 TypedArray로 준비하고, buildClassPalette 결과를 PointLayer에 넘기는 것입니다. 

import { useMemo, useRef } from "react";
import {
  DEFAULT_POINT_COLOR,
  PointLayer,
  buildClassPalette,
  hexToRgba,
  type PointHoverEvent,
  type PointQueryHandle,
  type PointSizeByZoom,
  type WsiClass,
  type WsiPointData,
} from "open-plant";

function CellLayer({
  classes,
  positions,
  paletteIndices,
}: {
  classes: WsiClass[];
  positions: Float32Array;
  paletteIndices: Uint16Array;
}) {
  const pointRef = useRef<PointQueryHandle>(null);
  const palette = useMemo(() => buildClassPalette(classes), [classes]);

  const pointData: WsiPointData = {
    count: positions.length / 2,
    positions,
    paletteIndices,
  };

  const sizeByZoom: PointSizeByZoom = { 1.25: 2, 2.5: 3, 5: 4, 10: 5, 20: 7 };

  console.log(DEFAULT_POINT_COLOR, hexToRgba("#13bfa0"));

  return (
    <PointLayer
      ref={pointRef}
      data={pointData}
      palette={palette.colors}
      sizeByZoom={sizeByZoom}
      strokeScale={1.15}
      innerFillOpacity={0.2}
      onHover={(event: PointHoverEvent) => console.log(event.id, event.coordinate)}
      onClick={(event) => console.log(event.index, event.id)}
    />
  );
}

const hit = pointRef.current?.queryAt([12000, 18000]);
간단한 좌표 조회가 목적이면 PointQueryHandle 또는 onHover/onClick이면 충분합니다. 직접 spatial hash를 다뤄야 할 때만 아래의 Point Hit Index 섹션으로 내려가세요.

4. DrawingLayer / RegionLayer / PatchLayer / OverlayLayer

ROI를 사용자 입력으로 만들 때는 DrawingLayer가 가장 일반적입니다. 이미 좌표를 갖고 있다면 RegionLayer, PatchLayer, OverlayLayer만으로도 충분합니다.

function StandaloneDrawOverlay() {

5. HeatmapLayer / OverviewMap / useViewerContext

hotspot 표현은 HeatmapLayer, 미니맵과 좌표 변환은 OverviewMap, useViewerContext 조합으로 붙입니다. 

import {
  HeatmapLayer,
  OverviewMap,
  useViewerContext,
  type HeatmapKernelScaleMode,
  type HeatmapPointData,
  type OverviewMapOptions,
} from "open-plant";

const heatmapData: HeatmapPointData = {
  count: hotspots.length / 2,
  positions: hotspots,
  weights,
};

const scaleMode: HeatmapKernelScaleMode = "fixed-zoom";

<HeatmapLayer
  data={heatmapData}
  visible
  opacity={0.75}
  radius={2}
  blur={3}
  gradient={["transparent", "#0ff", "#0f0", "#ff0", "#f00"]}
  scaleMode={scaleMode}
  fixedZoom={6}
  densityContrast={4}
  onStats={(stats) => console.log(stats.visiblePoints, stats.renderMs)}
/>
function ViewerChrome() {
  const { rendererRef, source, worldToScreen, screenToWorld, overviewInvalidateRef } = useViewerContext();
  if (!source) return null;

  const options: Partial<OverviewMapOptions> = {
    position: "bottom-right",
    viewportBorderStyle: "dash",
    interactive: true,
  };

  console.log(worldToScreen(1000, 1000), screenToWorld(200, 120));

  return (
    <OverviewMap
      source={source}
      projectorRef={rendererRef}
      invalidateRef={overviewInvalidateRef}
      options={options}
    />
  );
}
__heatmapLayerInternals는 internal/debug 용도입니다. 앱 공개 코드의 기본 경로로 잡지 않는 편이 안전합니다.

6. filterPointDataByPolygons*: sync / worker / hybrid-webgpu

포인트를 ROI 내부로 줄이고 싶다면 직접 함수 호출로 전처리할 수도 있고, PointLayer에 clipEnabled / clipMode를 넘겨 런타임에서 처리할 수도 있습니다. 

import {
  filterPointDataByPolygons,
  filterPointDataByPolygonsHybrid,
  filterPointDataByPolygonsInWorker,
  filterPointIndicesByPolygons,
  filterPointIndicesByPolygonsInWorker,
  terminateRoiClipWorker,
  type HybridPointClipOptions,
  type PointClipMode,
  type RoiPolygon,
} from "open-plant";

const polygons: RoiPolygon[] = [roi.coordinates as RoiPolygon];

const syncData = filterPointDataByPolygons(pointData, polygons);
const syncIndices = filterPointIndicesByPolygons(pointData, polygons);

const workerResult = await filterPointDataByPolygonsInWorker(pointData, polygons);
const workerIndices = await filterPointIndicesByPolygonsInWorker(pointData, polygons);

const hybridOptions: HybridPointClipOptions = { bridgeToDraw: true };
const hybridResult = await filterPointDataByPolygonsHybrid(pointData, polygons, hybridOptions);

const clipMode: PointClipMode = "worker";
terminateRoiClipWorker();
<PointLayer
  data={pointData}
  palette={palette.colors}
  clipEnabled
  clipToRegions={regions}
  clipMode="worker"
  onClipStats={(event) => {
    console.log(event.mode, event.durationMs, event.outputCount);
  }}
/>
모드권장 상황비고
"sync"작은 데이터, 단순 디버깅메인 스레드에서 바로 계산.
"worker"운영 기본값메인 UI 블로킹을 줄이기 가장 쉽습니다.
"hybrid-webgpu"실데이터 벤치마크 후 opt-inbbox prefilter 뒤 polygon 판정을 이어갑니다.
RoiClipWorkerRequest 계열 protocol 타입은 custom worker bridge를 직접 만들 때만 필요합니다. 일반 앱에서는 filterPointDataByPolygonsInWorker만 호출하면 됩니다.

7. buildPointSpatialIndexAsync / lookupCellIndex

hover/click은 PointLayer 기본 이벤트로 충분한 경우가 많습니다. 하지만 좌표 기반 질의를 직접 제어하거나 custom picking을 만들 때는 buildPointSpatialIndexAsync를 사용합니다. 

import {
  buildPointSpatialIndexAsync,
  lookupCellIndex,
  terminatePointHitIndexWorker,
  type FlatPointSpatialIndex,
} from "open-plant";

const flatIndex: FlatPointSpatialIndex | null = await buildPointSpatialIndexAsync(pointData, source);

if (flatIndex) {
  const cellIndex = lookupCellIndex(flatIndex, 12, 8);
  console.log(flatIndex.cellSize, flatIndex.safeCount, cellIndex);
}

terminatePointHitIndexWorker();
worker protocol 타입인 PointHitIndexWorkerRequest / PointHitIndexWorkerResponse는 내부 worker 통신 contract를 타입으로 맞추고 싶을 때만 직접 사용하면 됩니다.

8. parseWkt / toRoiGeometry / computeRoiPointGroups / createSpatialIndex

문자열 WKT를 ROI 좌표로 바꾸고, ROI별 class count를 계산하고, bbox 검색용 spatial index를 두는 흐름입니다. 

import {
  computeRoiPointGroups,
  createSpatialIndex,
  parseWkt,
  toRoiGeometry,
  type RoiPointGroupStats,
  type SpatialExtent,
  type SpatialIndexItem,
} from "open-plant";

const parsed = parseWkt("POLYGON ((1200 1400, 2400 1400, 2400 2600, 1200 2600, 1200 1400))");
const roiGeometry = toRoiGeometry(parsed?.coordinates);

const stats: RoiPointGroupStats = computeRoiPointGroups(pointData, regions, {
  paletteIndexToClassId: new Map([
    [1, "negative"],
    [2, "positive"],
  ]),
});

const index = createSpatialIndex<{ id: string }>(16);
const items: SpatialIndexItem<{ id: string }>[] = [
  { minX: 1200, minY: 1400, maxX: 2400, maxY: 2600, value: { id: "roi-1" } },
  { minX: 2600, minY: 3000, maxX: 4200, maxY: 4600, value: { id: "roi-2" } },
];
index.load(items);

const searchExtent: SpatialExtent = [1800, 1800, 3200, 3200];
const candidates = index.search(searchExtent);
RoiPointGroup, RoiClassCount, ParsedWktGeometry, ParsedWktPolygon, ParsedWktMultiPolygon은 이 흐름의 결과 타입들입니다.

10. TileViewerCanvas / M1TileRenderer / WsiTileRenderer
